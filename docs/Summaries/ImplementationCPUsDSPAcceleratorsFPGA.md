# Implementation of CPUs and DSP Accelerators on FPGA Summaries

### Design and Implementation of a RISC-V SoC with Custom DSP Accelerators for Edge Computing

- **Citation:** Yadav, Priyanshu. "Design and Implementation of a RISC-V SoC with Custom DSP Accelerators for Edge Computing." arXiv preprint arXiv:2506.06693 (2025).
- **Description:** This article explains how a simple 32-bit RISC-V CPU can be improved by adding special hardware units, called DSP accelerators, to make it faster and more efficient for tasks like signal processing and small-scale AI. RISC-V is an open-source processor design, meaning anyone can use and modify it for free. The paper focuses on creating a System-on-Chip (SoC) that includes a basic RISC-V processor along with two custom accelerators — one for 1D convolution (a common operation in filters and neural networks) and one for dot products (used in math and AI computations). Normally, the CPU would take many cycles to perform these repetitive calculations, but the accelerators can handle them in hardware much faster, reducing the processing time by about three times and using less energy. The paper also describes how these parts connect together through memory and control signals, and it discusses possible future improvements, such as adding more accelerators (like for FFT or matrix math), testing on real FPGA hardware, and creating supporting software tools. In simple terms, the article shows that instead of building a CPU just to run general code, you can make it far more powerful by adding custom “helper circuits” designed to handle specific, heavy-duty tasks efficiently.
